# CRISP-DM Phase 1: 業務理解 (Business Understanding)

## 1. 專案背景與目標

### 1.1 業務目標 (Business Objectives)
- **主要目標**: 開發基於HOSVD高階張量分解的手寫數字識別系統
- **學術應用**: 中興大學資訊系統研究所 - 數據分析數學課程作業
- **實務應用**: 證明HOSVD在高維圖像數據維度約減與分類中的有效性

### 1.2 專案背景
```

作業: Homework 2 - HOSVD 手寫辨識

```

### 1.3 成功標準 (Success Criteria)

| 指標 | 目標值 | 說明 |
|-----|-------|------|
| 整體分類準確率 | ≥ 95% | 對所有測試樣本的準確率 |
| 平均精確率 (P@) | ≥ 93% | 各類別精確率平均值 |
| 平均召回率 (R@) | ≥ 93% | 各類別召回率平均值 |
| 平均F1分數 | ≥ 94% | 精確率與召回率的調和平均 |
| HOSVD維度約減率 | ≥ 90% | 特徵維度減少率 |
| 算法運行時間 | < 30s | 完整訓練-測試循環時間 |

## 2. 關鍵問題定義 (Problem Definition)

### 2.1 核心問題
**如何利用高階張量分解（HOSVD）結合多分類器系統，高效準確地識別手寫數字？**

### 2.2 技術難點
1. **多維數據處理**: 28×28像素圖像轉換為張量結構
2. **維度約減**: 保留有效信息前提下進行維度約減（784 → ~50維）
3. **分類器選擇**: 在KNN、SVM、RF、MLP中選擇最優組合
4. **計算效率**: 張量分解計算複雜度優化

## 3. 資源與限制

### 3.1 可用資源 (Resources)
- **數據源**: 
  - MNIST數據集 (70,000張圖像，28×28像素，灰度)
  - Fashion-MNIST (70,000張時裝圖像)
  - USPS數據集 (9,298張郵政編碼)
  - scikit-learn內置數字數據集 (1,797張)

- **計算環境**:
  - Python 3.8+
  - 科學計算堆棧: NumPy, SciPy
  - 機器學習框架: scikit-learn
  - 張量計算: TensorLy

- **開發工具**:
  - Jupyter Notebook (互動式分析)
  - Matplotlib/Seaborn (數據可視化)
  - Git版本控制

### 3.2 時間與預算限制
- **交付截止**: 2025年春季學期末
- **預算**: 學術/開源軟件（零成本）
- **計算資源**: 標準CPU (GPU可選)

## 5. 假設與風險評估

### 5.1 關鍵假設 (Assumptions)
1. HOSVD確實能有效降低手寫數字圖像維度
2. 多分類器集成優於單一分類器
3. 訓練數據分布代表測試環境
4. 現有庫函數實現正確

### 5.2 風險因素 (Risks)

| 風險 | 可能性 | 影響 | 緩解方案 |
|-----|--------|------|---------|
| 維度約減過度導致精度下降 | 中 | 高 | 調整n_components參數 |
| 計算時間超時 | 低 | 中 | 實現矩陣緩存、並行計算 |
| 數據不平衡 | 低 | 中 | 加權分類、數據增強 |
| 庫版本不相容 | 低 | 低 | 凍結requirements.txt版本 |

## 7. 方法論與技術路線

### 7.1 選定方法：CRISP-DM
採用標準數據挖掘六階段模型：
1. **業務理解** ← 當前階段
2. **數據理解** → 分析MNIST特徵分布
3. **數據準備** → 張量化、正規化
4. **建模** → HOSVD + 多分類器
5. **評估** → 性能測試與對比
6. **部署** → CLI系統、模型保存

### 7.2 技術路線

```
原始圖像 (28×28)
    ↓
  [張量化] 28×28×1張量
    ↓
  [HOSVD] 核張量 (rank1 × rank2 × rank3)
    ↓
[特徵展平] 1D特徵向量
    ↓
[分類器] KNN/SVM/RF/MLP
    ↓
  [決策] 類別預測
```
